# Portfolio Benchmark — Data Analyst / BI

A short competitive read: what strong Data Analyst / BI portfolios do, and the concrete gaps
to close on `arabinda07.github.io`. Scope is *benchmarking for ideas* — not building
comparison pages.

_Last updated: 2026-06-27_

---

## Benchmarks reviewed

| Portfolio / source | What it does well |
|---|---|
| **Naledi Hollbruegge** (naledianalytics) | Clean, content-rich layout; a real "Portfolio" section of self-contained case studies across industries; testimonials. |
| **Alex The Analyst** | Project-led; each project has a clear problem → data → tools → result write-up and links to code + dashboard. |
| **datascienceportfol.io** templates | Standardised structure recruiters expect: project card → live demo link → GitHub repo → tech stack. |
| **Maggie Wolff / Kelly J. Adams** | Narrative case studies that show the *thinking* (what was measured and why), not just the chart. |
| **Vidi-corp / Power BI showcases** | Lead with the **dashboard image itself** as the hero visual; interactive "Publish to Web" embeds. |

Common thread across all of them: **visible, interactive dashboard proof + a documented
process (sourced → cleaned → modelled → visualised)**, plus links to code.

---

## Where this site already wins (ahead of most)

- **SEO / AI-SEO hygiene** — JSON-LD `@graph` (Person + WebSite + ProfilePage + ItemList),
  `llms.txt`, `profile.md`, AI-crawler-friendly `robots.txt`, sitemap. Most DA portfolios have
  *none* of this. This is a genuine, rare advantage for AI-answer citation.
- **Craft** — fast, accessible, responsive, single-H1, semantic, WebP/responsive images,
  deferred fonts. Cleaner than the average Wix/Squarespace DA site.
- **Voice** — strong, human narrative ("make the work easier to see"). Most portfolios read
  like résumés.

---

## Gaps to close (highest impact first)

1. **No visible dashboard proof — this is the biggest one.** The positioning is "Data Analyst
   & BI Consultant", but the Work section's visuals are *school-program event photos*, and the
   flagship LKS dashboard work is text-only ("functional view only; internal data protected").
   Recruiters skim for a dashboard they can *see*. **Fix:** build 1–2 public sample dashboards
   on **synthetic data** (Power BI Publish-to-Web embed or high-res screenshots) — e.g. a
   performance-review dashboard mirroring the LKS work's structure without any real data. This
   also adds indexable, AI-citable content.

2. **No per-project process breakdown.** Strong portfolios walk through sourced → cleaned →
   modelled → dashboard. This site states outcomes but not the analytical path. **Fix:** add a
   short "How it was built" line/list to the featured BI projects (SQL → Python/Pandas →
   Power BI), which the experience bullets already hint at — surface it in the Work cards too.

3. **No code links on analytics work.** GitHub is linked at the profile level, but the
   SQL/Python featured work has no repo. Internal LKS work can't be shared — but a **public
   sample-analysis repo** (the synthetic dashboard's queries + notebook) would give the
   "show me the code" proof recruiters look for.

4. **No testimonials / third-party proof.** Every top benchmark has at least one. **Fix:** one
   or two short quotes (manager, teacher, collaborator) — also strengthens E-E-A-T for SEO.

5. **Dashboard imagery as the hero of the Work section.** Lead BI projects with a dashboard
   thumbnail rather than program photos, so the first visual matches the "BI" claim.

---

## Recommended next 3 actions

1. Ship **one public synthetic-data Power BI dashboard** + embed/screenshot in the Work
   section. (Closes gaps 1, 2, 5 at once; biggest credibility jump.)
2. Add a **public sample repo** with that dashboard's SQL + Python, linked from the card.
3. Add **one testimonial**.

Everything else on the site (SEO, structure, copy, performance) is already at or above
benchmark — the missing piece is *seeable, clickable BI proof*.
